# TypeScript Best Practices Skill

This skill helps agents generate and review high-quality TypeScript code with a strong focus on:

- Type safety and strict typing
- Immutability and predictable data flow
- Typed error handling with `Result`-style patterns
- SOLID-oriented design and dependency boundaries
- Practical class design (`static` and `abstract` usage)

## When to use it

Use this skill for:

- Creating or refactoring TypeScript modules/services
- Reviewing TypeScript code quality and maintainability
- Applying architecture and API design patterns
- Answering TypeScript best-practice questions

## What it includes

- Main guidance: `SKILL.md`
- Deep-dive references under `references/`:
  - `type-system/`
  - `patterns/`
  - `anti-patterns/`
  - `architecture/`

## Scope note

This skill is TypeScript-focused and intentionally framework-agnostic.
